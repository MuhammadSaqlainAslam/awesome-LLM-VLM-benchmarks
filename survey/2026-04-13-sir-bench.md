# SIR-Bench: Evaluating Investigation Depth in Security Incident Response Agents (2026)

## Problem
Security incident response (IR) agents are evaluated almost exclusively on whether they reach the correct conclusion (true/false positive classification), ignoring whether the agent actually performed genuine forensic investigation to arrive at that conclusion. An agent that memorizes alert patterns can achieve high triage accuracy without discovering any evidence — a dangerous failure mode in production security operations where missed forensic findings enable attacker persistence. No benchmark existed to measure investigative depth as distinct from outcome accuracy.

## Method
**SIR-Bench** (arXiv: 2604.12040, April 13, 2026) introduces a **794-test-case** framework that evaluates autonomous security incident response agents using three complementary metrics: **triage accuracy** (correct positive/negative classification), **novel finding discovery** (number of previously unknown key findings surfaced), and **tool usage appropriateness** (whether the right forensic tools were used in the right order). The benchmark uses the **Once Upon A Threat (OUAT)** framework to recreate real incident scenarios in controlled cloud environments from actual IR case histories. A distinctive evaluation mechanism deploys an **adversarial LLM-as-Judge** that requires concrete forensic evidence to credit any finding — preventing shallow pattern-matching from being rewarded as genuine investigation.

Authors: Daniel Begimher, Cristian Leo, Jack Huang, Pat Gaw, Bonan Zheng.

## Benchmarks / Datasets
- **794 test cases** across diverse incident types (malware, lateral movement, data exfiltration, insider threat)
- Once Upon A Threat (OUAT) framework: real scenario recreation in controlled cloud
- Three evaluation dimensions: triage accuracy, novel finding discovery, tool appropriateness
- Adversarial LLM-as-Judge requiring concrete forensic evidence
- **Metrics:** True positive detection rate; false positive rejection rate; avg novel findings per case; tool appropriateness score

## Key Results

| Metric | Score | Notes |
|---|---|---|
| True positive detection rate | **97.1%** | High triage accuracy |
| False positive rejection rate | **73.4%** | Meaningful false alarm reduction |
| Avg novel key findings / case | **5.67** | Investigative depth measure |

- **97.1% true positive detection rate** with **73.4% false positive rejection** — triage accuracy is strong for the evaluated agent
- **5.67 novel key findings per case on average** — the agent surfaces previously unlogged forensic evidence beyond the triggering alert, the core contribution of the benchmark
- The adversarial judge design successfully filters surface-level pattern matching: agents must produce verifiable evidence chains, not just correct labels
- The OUAT scenario recreation framework enables reproducible incident response evaluation without exposing live production environments

## FoxBrain Relevance
SIR-Bench is directly applicable to Foxconn's OT/IT security operations, which face nation-state and supply-chain attack vectors against manufacturing infrastructure. The 97.1% triage accuracy is encouraging for AI-assisted incident triage, but the novel-findings metric is the real value signal: FoxBrain IR agents should be tested on their ability to surface attacker lateral movement paths and data staging evidence, not merely detect known-bad indicators. The OUAT framework can be adapted to recreate Foxconn-relevant threat scenarios (ICS/SCADA attacks, supply chain compromises) for internal red team exercises. The adversarial judge design should be adopted as FoxBrain's IR evaluation standard to prevent false confidence from high-accuracy-but-shallow agents.
