# SciCode: A Research Coding Benchmark Curated by Scientists (2024)

## Problem
Existing coding benchmarks (HumanEval, MBPP, LeetCode) test algorithmic or competitive programming skills that bear little resemblance to **scientific research coding** — which requires deep domain knowledge, the ability to translate physical or mathematical equations into correct implementations, and reproducing results from published papers. No benchmark evaluated LLMs on the kind of code actually written by scientists in research workflows.

## Method
Scientists across 5 natural science fields curated real research problems directly from their own workflows and published papers. Each main problem is decomposed into sequential subproblems to enable fine-grained diagnosis. Two evaluation settings are used: **(1) with background scientific context provided** (background document accompanying the problem) and **(2) without** (more realistic). Solutions from prior subproblems can optionally be provided to test isolated subproblem-solving ability. Evaluation uses scientist-annotated gold-standard solutions with custom test cases. Published at **NeurIPS 2024 Datasets and Benchmarks Track**, with contributors from MIT, Harvard, Stanford, Princeton, University of Chicago, UIUC, and Argonne National Laboratory.

## Benchmarks / Datasets
- **80 main problems → 338 subproblems** (Dev: 15 main / 50 sub; Test: 65 main / 288 sub)
- **5 natural science domains, 16 subdomains:**
  - **Physics (37 problems):** Condensed Matter, Optics, Quantum Computing, Astrophysics, Particle Physics, Computational Physics
  - **Mathematics (14 problems):** Numerical Linear Algebra, Computational Mechanics, Computational Finance
  - **Material Science (13 problems):** Semiconductor Materials, Molecular Modeling
  - **Chemistry (8 problems):** Quantum Chemistry, Computational Chemistry
  - **Biology (8 problems):** Ecology, Biochemistry, Genetics
- Problems sourced directly from active scientists' research and published papers

## Key Results (Standard Setting — no background context, using generated prior subproblem solutions)

| Model | Subproblem Pass@1 | Main Problem Pass@1 |
|---|---|---|
| **Claude 3.5 Sonnet** | **26.0%** | **4.6%** |
| GPT-4o | 25.0% | 1.5% |
| GPT-4-Turbo | 22.9% | 1.5% |
| Gemini 1.5 Pro | 21.9% | 1.5% |
| Claude 3 Opus | 21.5% | 1.5% |
| DeepSeek-Coder-v2 | 21.2% | 3.1% |
| Llama-3-70B-Instruct | 14.6% | 0.0% |

All frontier models achieve **below 5% on main problem pass@1** — meaning end-to-end scientific research coding remains almost entirely unsolved by current LLMs. Claude 3.5 Sonnet leads but at only 4.6%. The gap between subproblem accuracy (~26%) and main problem accuracy (~4.6%) reveals that errors compound rapidly across the sequential steps of a real research workflow.

## FoxBrain Relevance
SciCode's Physics domain (37 problems) — especially Condensed Matter, Optics, and Computational Physics — maps directly to Foxconn's materials science, semiconductor process, and optical inspection R&D workflows. The catastrophically low main-problem pass@1 scores (~4.6% even for the best model) indicate that FoxBrain cannot yet be trusted for end-to-end autonomous scientific code generation in research contexts — human expert review is mandatory at every step. The subproblem decomposition evaluation strategy provides a blueprint for FoxBrain's internal R&D coding assessment: stage-wise validation at each computational step before integrating into a full research pipeline.
