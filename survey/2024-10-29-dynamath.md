# DynaMath: A Dynamic Visual Benchmark for Evaluating Mathematical Reasoning Robustness of Vision Language Models (2024)

## Problem
Standard VLM math benchmarks use fixed problem sets, so average accuracy can mask brittleness: a model may solve a problem in one visual form but fail on a trivially varied version (different numerical values, axis labels, or layout). High average scores thus overstate true mathematical reasoning ability. DynaMath exposes this gap by evaluating robustness — the ability to answer correctly across multiple visual variants of the same problem, not just the canonical form.

## Method
DynaMath encodes 501 high-quality seed questions as **Python programs** that auto-generate visual and textual variations. Each seed yields 10 concrete variants (5,010 total problems), covering changes in numerical values, graph structures, coordinate systems, figure layouts, and distractors. Nine mathematical topic areas span Solid Geometry (3.0%), Puzzle Tests (3.4%), Arithmetic (5.2%), Scientific Figures (9.0%), Graph Theory (9.6%), Algebra (10.2%), Plane Geometry (15.4%), Analytic Geometry (19.4%), and Statistics (25.0%). The key metric is **worst-case accuracy** — the fraction of seed questions answered correctly on all 10 variants — providing a strict lower bound on reasoning reliability. Fourteen VLMs are evaluated alongside a human baseline.

## Benchmarks / Datasets
- **DynaMath:** 501 seed questions × 10 variants = 5,010 total problems, 9 topic areas
- **Human baseline:** 77.3% average accuracy across all problems
- **14 VLMs evaluated:** Claude-3.5 Sonnet, GPT-4o, Gemini Pro 1.5, Qwen2-VL-72B, InternVL2-76B, Llama 3.2-90B, and others

## Key Results
| Model | Avg Accuracy | Worst-Case Accuracy |
|---|---|---|
| **Claude-3.5 Sonnet** | 64.8% | 35.3% |
| **GPT-4o** | 63.7% | 34.7% |
| **Gemini Pro 1.5** | 60.5% | 26.9% |
| **Qwen2-VL-72B** | 55.1% | 28.3% |
| **InternVL2-76B** | 54.0% | 24.6% |
| **Llama 3.2-90B** | 44.0% | 13.0% |
| **Human** | **77.3%** | — |

- All models show dramatic drops from average to worst-case accuracy (e.g., GPT-4o: 63.7% → 34.7%)
- Human performance is consistent across variants; model performance is not — revealing genuine pattern-matching rather than principled reasoning
- First dynamic visual math benchmark using programmatic seed generation

## FoxBrain Relevance
DynaMath's worst-case accuracy metric is a direct measure of engineering-critical reliability — for FoxBrain deployments on measurement data, manufacturing charts, or process control graphs, a model that fails on minor visual perturbations (axis rescaling, unit changes, color shifts) creates unacceptable production risk. FoxBrain should be benchmarked on DynaMath's Analytic Geometry and Statistics categories before any deployment involving numerical chart interpretation, and worst-case accuracy (not average accuracy) should be the gating metric.
