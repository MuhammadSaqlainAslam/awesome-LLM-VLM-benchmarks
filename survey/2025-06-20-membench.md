# MemBench: Towards More Comprehensive Evaluation on the Memory of LLM-based Agents (2025)

## Problem
LLM-based agent evaluation frameworks largely overlook memory as a first-class capability, conflating it with general QA performance. Without a structured benchmark separating factual memory, reflective memory, efficiency, and capacity dimensions, it is impossible to diagnose or compare memory subsystems across different agent architectures.

## Method
MemBench evaluates agent memory across three dimensions — effectiveness, efficiency, and capacity — and two interaction scenarios: participation (the agent is an active participant) and observation (the agent passively observes). It introduces factual memory (verbatim retention) and reflective memory (synthesis across sessions) as distinct evaluation levels, using ~500 user profile graphs and 65,000 sessions to generate 53,000 questions. Published in ACL 2025 Findings.

## Benchmarks / Datasets
- **MemBench:** ~53,000 questions across 65,000 sessions/trajectories; 500 user profile graphs.
- **Dimensions:** Effectiveness (accuracy/recall), Efficiency (temporal), Capacity (volume limits).
- **Scenarios:** Participation vs. Observation; Factual vs. Reflective memory.
- **Metrics:** Accuracy, recall rate, capacity score, temporal efficiency.
- **GitHub:** https://github.com/import-myself/Membench

## Key Results
All evaluated LLMs show significant degradation in reflective memory compared to factual memory, and observation scenarios prove harder than participation. Memory capacity drops sharply as session count exceeds a model-specific threshold, revealing that current agent memory architectures are not designed for long-horizon tasks.

## FoxBrain Relevance
Critical for any FoxBrain deployment involving persistent user sessions or long-running industrial workflows. The factual vs. reflective memory split is a practical diagnostic: FoxBrain's agent memory should be tested on both before deploying in customer-facing or process-monitoring contexts. The capacity dimension directly informs how many sessions can be maintained reliably.
