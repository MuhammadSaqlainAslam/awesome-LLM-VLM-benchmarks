# ENAMEL: How Efficient is LLM-Generated Code? A Rigorous & High-Standard Benchmark (2024)

## Problem
Code generation benchmarks (HumanEval, MBPP) measure only functional correctness, not algorithmic efficiency. An LLM that produces a correct O(n²) solution where O(n log n) is possible is not production-ready — but existing benchmarks cannot distinguish these cases, creating a blind spot in LLM coding capability assessment.

## Method
ENAMEL (EfficeNcy AutoMatic EvaLuator) benchmarks LLMs specifically on code efficiency. Human experts wrote best-algorithm reference solutions and test case generators for each problem. The benchmark introduces a new metric `eff@k` that generalizes `pass@k` from correctness to efficiency — measuring whether at least one of k generated solutions achieves expert-level runtime. 30 LLMs are evaluated and ranked. Published at ICLR 2025.

## Benchmarks / Datasets
- **ENAMEL:** Two subsets; expert-written reference solutions; automated test case generators.
- **Metric:** `eff@k` (efficiency analog of pass@k); also reports correctness vs. efficiency jointly.
- **Models Evaluated:** 30 LLMs (full leaderboard).
- **GitHub:** https://github.com/q-rz/enamel

## Key Results
**All 30 evaluated LLMs fall short of expert-level efficient code.** The top models generate correct solutions reliably but consistently choose suboptimal algorithms — typically O(n²) where O(n log n) or O(n) solutions exist. The gap between correctness performance and efficiency performance is larger for harder algorithmic problems.

## FoxBrain Relevance
High relevance for FoxBrain's code generation and automation pipelines. If FoxBrain generates code for data processing, simulation, or optimization tasks in manufacturing or logistics, functional correctness alone is insufficient — runtime efficiency directly impacts operational costs. ENAMEL's `eff@k` metric should be adopted as a standard evaluation layer in FoxBrain's code generation evaluation suite.
