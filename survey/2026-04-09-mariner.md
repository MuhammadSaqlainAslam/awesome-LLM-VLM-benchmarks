# MARINER: A 3E-Driven Benchmark for Fine-Grained Perception and Reasoning in Maritime VLMs (2026)

## Problem
Maritime domains — ports, open water, coastal surveillance, vessel traffic management — represent a high-stakes VLM application where fine-grained visual understanding and causal event reasoning are both critical. Standard VLM benchmarks use generic RGB scenes; no benchmark evaluated VLMs on the specific challenges of maritime imagery: fine-grained vessel classification across dozens of categories, multi-party incident reasoning, and dynamic event understanding under challenging visual conditions (glare, occlusion, distance, weather). This gap left maritime AI deployments without a principled capability evaluation framework.

## Method
**MARINER** (arXiv: 2604.08615, April 9, 2026) introduces a **3E (Entity-Environment-Event) paradigm** for maritime visual understanding, evaluated on **16,629 multi-source maritime images** across three capability tracks:

- **Entity Track:** Fine-grained vessel classification across **63 vessel categories** (cargo ships, tankers, fishing vessels, ferries, naval vessels, etc.) — tests whether VLMs can distinguish visually similar vessel types at expert level
- **Environment Track:** Object detection and spatial relationship inference in complex maritime environments (harbours, open sea, port facilities, coastal infrastructure)
- **Event Track:** Visual QA across **5 dynamic maritime incident types** (collision, near-miss, grounding, oil spill, distress situation) — tests causal attribution and temporal event understanding

Authors: Xingming Liao, Ning Chen, Muying Shu, Yunpeng Yin, Peijian Zeng, Zhuowei Wang, Nankai Lin, Lianglun Cheng.

## Benchmarks / Datasets
- **16,629 multi-source maritime images**
- **63 vessel categories** for fine-grained classification
- **5 dynamic maritime incident types** for event VQA
- **3E paradigm:** Entity (vessel classification) + Environment (detection/spatial) + Event (incident VQA)
- **Metrics:** Classification accuracy, detection mAP, VQA accuracy per incident type

## Key Results

| Track | Model Performance | Key Challenge |
|---|---|---|
| **Entity (vessel classification)** | All models struggle on fine-grained sub-categories | 63-class distinction requires expert-level visual discrimination |
| **Environment (detection)** | Better performance on prominent objects | Small/distant vessels problematic |
| **Event (incident VQA)** | Weakest track across all models | Causal attribution near-chance |

- **Fine-grained vessel classification is the hardest Entity task** — models easily classify generic "ship" vs. "boat" but fail to distinguish cargo ship sub-types, naval vessel classes, or fishing boat categories at operational precision
- **Maritime incident causal attribution mirrors CrashSight's findings** — models describe incident scenes adequately but cannot reliably identify fault, cause, or temporal event sequence
- The 3E paradigm successfully reveals that VLM strengths (environmental context) and weaknesses (fine-grained entity classification, causal event reasoning) are domain-consistent with general VLM evaluations

## FoxBrain Relevance
MARINER's 3E paradigm is directly adaptable for Foxconn's manufacturing visual inspection challenges: the Entity track (fine-grained vessel classification) maps onto fine-grained component classification (PCB component types, connector variants, substrate materials); the Environment track maps onto spatial assembly verification (component placement, orientation, spatial relationships); the Event track maps onto defect event analysis (how/when a defect developed). The benchmark's finding that VLMs struggle with fine-grained entity classification is a critical calibration point for FoxBrain's inspection pipeline — the model needs domain-specific fine-tuning on Foxconn's specific component taxonomy before reliable fine-grained classification in production.
