# DISSECT: Diagnosing VLM Failures — Perception vs. Language-Prior Reasoning in Scientific Tasks (2026)

## Problem
When a Vision-Language Model fails on a scientific problem requiring image understanding, there are two fundamentally different root causes: (1) the model cannot correctly *perceive* the relevant visual information from the image, or (2) the model correctly perceives the image but fails to *reason* correctly with that information, instead falling back on language-prior shortcuts that bypass genuine visual analysis. These two failure modes require entirely different interventions — better visual encoders vs. better reasoning training — yet no benchmark systematically decomposed VLM science failures into these two distinct categories. Teams optimising VLMs for scientific tasks were treating perception and reasoning failures as a single undifferentiated problem.

## Method
**DISSECT** (arXiv: 2604.06250, April 6, 2026) introduces a **Model Oracle protocol** that decomposes VLM performance across five input modes to isolate perception and reasoning contributions:

1. **Vision+Text** — standard VLM input (baseline)
2. **Text-Only** — problem text without the image (measures language-prior exploitation)
3. **Vision-Only** — image without text (measures raw visual processing)
4. **Human Oracle** — human-written description of the image replaces the image (measures reasoning given perfect perception)
5. **Model Oracle** — the model's own description of the image replaces the image (measures reasoning given the model's own visual understanding)

The gap between Vision+Text and Human Oracle performance isolates **perception failures**. The gap between Human Oracle and text-only performance isolates **reasoning failures**.

**12,000 diagnostic questions** across two scientific domains:
- **7,000 Chemistry** — molecular structures, reaction diagrams, laboratory apparatus
- **5,000 Biology** — cell diagrams, microscopy images, anatomical charts

Authors: Dikshant Kukreja, Kshitij Sah, Karan Goyal, Mukesh Mohania, Vikram Goyal.

## Benchmarks / Datasets
- **12,000 diagnostic questions:** 7,000 Chemistry + 5,000 Biology
- **5 input modes** per question (Vision+Text, Text-Only, Vision-Only, Human Oracle, Model Oracle)
- **Metrics:** Perception fidelity score; integration effectiveness; language-prior exploitation score
- Both closed-source and open-source VLMs evaluated

## Key Results

| Finding | Detail |
|---|---|
| **Closed-source vs. open-source** | Closed-source outperforms on *integration* (perception + reasoning together) |
| **Open-source bottleneck** | Systematic failures when reasoning from raw images vs. own descriptions |
| **Language-prior exploitation** | All models score higher on Text-Only than Vision-Only — confirming language shortcuts |
| **Largest perception gap** | Biology microscopy images (most ambiguous visual content) |
| **Chemistry advantage** | Molecular structure perception stronger than biological diagram perception |

- **Open-source models show a systematic bottleneck**: performance on Model Oracle (model's own image description) is substantially higher than on Vision+Text — confirming the visual encoder, not reasoning, is the primary open-source failure point
- **All evaluated VLMs exploit language priors**: Text-Only performance exceeds Vision-Only performance, meaning models are partially answering from memorised text patterns rather than genuine visual analysis
- **Closed-source model integration effectiveness** is the primary advantage over open-source — they better combine visual and textual information

## FoxBrain Relevance
DISSECT's diagnostic framework is directly applicable to FoxBrain's visual inspection and scientific document understanding pipelines. For manufacturing QC tasks (defect detection, PCB inspection, component verification), it's critical to distinguish whether FoxBrain failures are due to visual encoder limitations (cannot perceive the defect) vs. reasoning failures (perceives but misclassifies). The Model Oracle protocol can be applied immediately to FoxBrain: run both Vision+Text and Model-Oracle conditions on the same inspection tasks to diagnose whether investing in better visual encoders vs. better reasoning training will yield the highest return. The finding that language priors dominate for all models is a critical warning for any FoxBrain deployment involving standardised document types — the model may be pattern-matching from training rather than reading the actual document.
