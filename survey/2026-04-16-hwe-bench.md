# HWE-Bench: Benchmarking LLM Agents on Real-World Hardware Bug Repair Tasks (2026)

## Problem
Existing LLM coding benchmarks focus almost exclusively on software (Python, JavaScript, etc.), leaving hardware description languages (Verilog, SystemVerilog, Chisel) entirely unevaluated at the repository level. Hardware debugging requires unique skills — fault localization across RTL, configuration, and verification files, plus hardware-semantic reasoning about signal propagation — that software benchmarks cannot measure.

## Method
**HWE-Bench** (arXiv: 2604.14709, April 16, 2026) introduces the first large-scale repository-level benchmark for evaluating LLM agents on practical hardware debugging. The dataset contains 417 task instances derived from real historical bug-fix pull requests across six major open-source projects (including RISC-V cores and complex SoC designs). Seven different LLMs are evaluated using four agent frameworks.

Authors: Fan Cui, Hongyuan Hou, Zizhang Luo, Chenyun Yin, Yun Liang

## Benchmarks / Datasets
- 417 task instances from real historical bug-fix pull requests across 6 open-source hardware projects
- Hardware languages: Verilog, SystemVerilog, Chisel
- 7 LLMs evaluated across 4 agent frameworks
- Projects range from simple RISC-V cores to complex SoC designs
- Metrics: task resolution rate, performance stratified by project complexity

## Key Results

| Project Type | Best Agent Performance | Key Challenge |
|---|---|---|
| Simple RISC-V cores | >90% tasks resolved | Well-structured, limited scope |
| Complex SoC projects | <65% tasks resolved | Cross-artifact coordination |
| Overall (best agent) | 70.7% | Fault localization bottleneck |
| Performance variance | Larger than SW benchmarks | Hardware semantics uniquely hard |

- **Best agent resolves 70.7% of hardware bug repair tasks overall**
- Performance exceeds 90% on simple RISC-V cores but drops below 65% on complex SoC projects
- Three major bottlenecks identified: fault localization, hardware-semantic reasoning, cross-artifact coordination

## FoxBrain Relevance
HWE-Bench is highly relevant to Foxconn's chip design and hardware engineering divisions, where engineers spend significant time debugging RTL designs for custom ICs used in consumer electronics and industrial control systems. The benchmark's finding that fault localization and cross-artifact coordination are the hardest subtasks directly informs where FoxBrain could provide the highest value in hardware development workflows — automated localization of likely fault sites before human engineers perform deep diagnosis. The 70.7% overall resolution rate demonstrates that LLM agents are already production-viable on simpler hardware debug tasks, making this an actionable near-term FoxBrain application.
