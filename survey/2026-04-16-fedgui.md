# FedGUI: Benchmarking Federated GUI Agents across Heterogeneous Platforms, Devices, and Operating Systems (2026)

## Problem
Existing GUI agent benchmarks evaluate agents in centralized settings where all training data is pooled together, ignoring real-world privacy constraints and platform heterogeneity. Deploying GUI agents at scale requires federated learning approaches that can handle cross-platform, cross-device, cross-OS, and cross-source heterogeneity — none of which prior benchmarks systematically address.

## Method
**FedGUI** (arXiv: 2604.14956, April 16, 2026) is the first comprehensive benchmark for developing and evaluating federated GUI agents across mobile, web, and desktop platforms. It provides six curated datasets examining four heterogeneity types: cross-platform, cross-device, cross-OS, and cross-source. The work was accepted as ACL 2026 Findings.

Authors: Wenhao Wang, Haoting Shi, Mengying Yuan, Yiquan Lin, Panrong Tong, Hanzhang Zhou, Guangyi Liu, Pengxiang Zhao, Yue Wang, Siheng Chen

## Benchmarks / Datasets
- 6 curated datasets spanning mobile, web, and desktop GUI environments
- 4 heterogeneity types evaluated: cross-platform, cross-device, cross-OS, cross-source
- Evaluation of federated learning strategies under each heterogeneity type
- Task success rate as the primary metric for GUI agent performance

## Key Results

| Heterogeneity Factor | Impact | Key Finding |
|---|---|---|
| Platform | High influence | Platform type is most influential factor in federated performance |
| Operating System | High influence | OS variation causes significant performance degradation |
| Cross-platform collaboration | Positive | Collaborative federated training yields measurable performance improvements |
| Privacy-preserving training | Feasible | Federated approach maintains competitive accuracy vs. centralized |

- **Cross-platform collaboration yields measurable performance improvements in federated GUI agent training**
- Platform and operating system are identified as the most influential heterogeneity factors
- FedGUI provides the first principled framework for privacy-preserving GUI agent evaluation at scale

## FoxBrain Relevance
FedGUI directly addresses Foxconn's enterprise privacy requirements — factory-floor GUI automation data cannot be centrally pooled across different facilities, customer projects, or operating systems. The benchmark's four heterogeneity dimensions (cross-platform, cross-device, cross-OS, cross-source) mirror real constraints in deploying FoxBrain GUI agents across Foxconn's global manufacturing facilities running diverse ERP systems (SAP, Oracle, custom MES). The finding that platform and OS are dominant heterogeneity factors provides actionable guidance for FoxBrain's federated training strategy. Adopting FedGUI's six-dataset evaluation framework would let Foxconn measure GUI agent robustness before cross-facility deployment.
