# VLR-Bench: Multilingual Benchmark for Vision-Language Retrieval Augmented Generation (2024)

## Problem
Existing VLM benchmarks evaluate models on closed-book visual QA, but real-world multimodal assistants increasingly need to retrieve and use **external documents** alongside visual inputs. No benchmark tested whether VLMs can perform reliable **Retrieval Augmented Generation (RAG)** in a visual context — or distinguish useful passages from distractors.

## Method
**VLR-Bench** is a VQA benchmark for evaluating VLMs on **vision-language RAG** tasks. Each problem is structured as `(image, query, 5 passages)`, where only **2 of 5 passages** contain directly relevant information. Models must identify which passages are useful and generate a grounded answer. A companion training dataset, **VLR-IF** (32,000 instruction-following examples), is also released to improve VLM RAG capabilities.

## Benchmarks / Datasets
* **VLR-Bench** — **300 VQA problems**, each with:
  * 1 image
  * 1 query requiring external knowledge
  * 5 candidate passages (2 relevant, 3 distractors)
* Multilingual coverage (EN, ZH, and other languages).
* **VLR-IF** — 32,000 instruction-following training samples for RAG fine-tuning.
* **Metric:** RAG Accuracy — whether the model selects the right passage(s) and answers correctly.
* Accepted at **COLING 2025**.

## Key Results
* Current VLMs perform significantly worse than expected when distractor passages are present, revealing a fundamental **passage selection failure** in multimodal RAG settings.
* Models fine-tuned on VLR-IF show substantial improvement in passage grounding (+10–15% accuracy vs. zero-shot).
* Multilingual results show consistent performance gaps for non-English queries, especially with image-text cross-modal RAG.

## FoxBrain Relevance
Critical for FoxBrain's **document-grounded visual assistant** use cases — e.g., retrieving product manuals, technical schematics, or compliance documents alongside user images. VLR-Bench validates whether the vision-RAG pipeline correctly grounds answers in retrieved content rather than hallucinating from memory.
