# The Amazing Agent Race (AAR): DAG-Structured Multi-Step Tool Navigation Benchmark (2026)

## Problem
Agent benchmarks typically evaluate linear task sequences — complete step A, then step B, then step C. But real-world knowledge work involves **directed acyclic graph (DAG) structures**: parallel information-gathering paths that must be completed and then aggregated into a final answer. No benchmark tested whether LLM agents could navigate DAG-structured workflows — where multiple independent sub-tasks must be completed across different tools, with results synthesised at convergence points — a structure common in research, competitive intelligence, and multi-source analysis tasks.

## Method
**The Amazing Agent Race (AAR)** (arXiv: 2604.10261, April 11, 2026) constructs **1,400 instances** based on **Wikipedia-grounded DAG puzzles** across 4 difficulty levels. Each puzzle consists of DAG "legs" — sub-tasks that must be completed by chaining Wikipedia lookups, calculator operations, and string manipulation tools — with answers aggregated at DAG convergence points. Two task structures are tested:
- **Sequential DAGs (800 instances):** Legs must be completed in order (each answer feeds the next)
- **Compositional DAGs (600 instances):** Multiple independent legs completed in any order, results composed at the end

Three **complementary metrics** diagnose distinct failure modes:
1. **Finish-line accuracy** — did the agent produce the correct final answer?
2. **Pit-stop visit rate** — did the agent complete all required intermediate steps?
3. **Roadblock completion rate** — did the agent successfully resolve blocking sub-tasks?

Authors: Zae Myung Kim, Dongseok Lee, Jaehyung Kim, Vipul Raheja.

## Benchmarks / Datasets
- **1,400 instances:** 800 sequential + 600 compositional DAG puzzles
- **4 difficulty levels** (number of DAG legs and depth)
- **Tools:** Wikipedia lookup, calculator, string manipulation
- **3 diagnostic metrics:** Finish-line accuracy, Pit-stop visit rate, Roadblock completion rate
- **Models evaluated:** Claude Code, Codex CLI, GPT-5.4, Gemini-3.1-Pro, and others

## Key Results

| Model | Finish-Line Accuracy | Token Efficiency | Notes |
|---|---|---|---|
| **Claude Code** | **37.2%** | **6× fewer tokens than Codex CLI** | Best efficiency |
| Codex CLI | ~37% | 6× more tokens | Same accuracy, far more expensive |
| GPT-5.4 | Below 37.2% | — | |
| Gemini-3.1-Pro | Below 37.2% | — | |

- **Claude Code achieves 37.2% finish-line accuracy** — matching Codex CLI's performance at 6× lower token consumption, establishing Claude Code as the most token-efficient agent for DAG navigation
- **All models fail significantly on compositional DAGs** relative to sequential DAGs — parallel path management and result aggregation are harder than linear chaining
- **Pit-stop visit rate reveals a hidden failure mode:** models sometimes produce correct final answers through incorrect intermediate steps (lucky guesses), and sometimes fail the finish line despite completing all pit stops (aggregation errors)
- Difficulty Level 4 (deepest DAGs) shows near-zero finish-line accuracy for all evaluated agents

## FoxBrain Relevance
AAR's DAG structure maps directly onto FoxBrain's most demanding enterprise workflows: supply chain analysis (multiple parallel supplier lookups → consolidated risk assessment), competitive intelligence (parallel product/pricing/patent research → synthesis), and multi-site quality correlation (parallel factory data collection → root cause aggregation). The 37.2% ceiling across all frontier models confirms that DAG-structured multi-step reasoning is an unsolved problem — FoxBrain should not be deployed on autonomous DAG workflows without explicit orchestration scaffolding that tracks intermediate state. Claude Code's 6× token efficiency advantage at the same accuracy level makes it the recommended backbone for FoxBrain's cost-sensitive multi-step agent workflows. The 3-metric diagnostic framework is immediately deployable as FoxBrain's agent evaluation suite.
