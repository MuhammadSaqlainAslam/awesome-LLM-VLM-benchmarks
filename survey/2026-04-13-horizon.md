# HORIZON: Diagnosing Where and Why Agentic Systems Break on Long-Horizon Tasks (2026)

## Problem
LLM agents appear to perform well on short and medium-horizon tasks, but their failure modes on long-horizon tasks are poorly understood. Existing evaluations report aggregate success/failure rates without diagnosing *where* in a long task chain the failure occurs, *why* it occurs (planning error, tool misuse, context loss, hallucination), or *how* failure modes compound across task length. Without structured failure attribution, agent developers cannot target improvements effectively.

## Method
**HORIZON** (arXiv: 2604.11978, April 13, 2026) introduces a **diagnostic framework** for analyzing long-horizon agent failures across multiple domains. The framework supports systematic construction of **escalating-complexity tasks** and provides structured **failure attribution analysis** — categorizing failures by type (planning, tool selection, context management, grounding) and by position in the task trajectory. Evaluation spans **3,100+ agent trajectories** across **four agentic domains**, testing state-of-the-art agents from GPT-5 and Claude model families. An LLM-as-a-Judge pipeline achieves strong inter-annotator agreement (κ=0.61) and high human alignment (κ=0.84), enabling scalable failure attribution without requiring human labeling of every trajectory. The **HORIZON Leaderboard** is released to enable community-driven comparative evaluation.

Authors: Xinyu Jessica Wang, Haoyue Bai, Yiyou Sun, Haorui Wang, Shuibai Zhang, Wenjie Hu, Mya Schroder, Bilge Mutlu, Dawn Song, Robert D Nowak.

## Benchmarks / Datasets
- **3,100+ agent trajectories** evaluated
- **4 agentic domains** with escalating complexity levels
- Tasks from short-horizon through long-horizon (controlled complexity ladder)
- GPT-5 and Claude family models evaluated
- LLM-as-a-Judge with κ=0.61 IAA and κ=0.84 human alignment
- HORIZON Leaderboard (public, community-extensible)
- **Metrics:** Task success rate by horizon length; failure type distribution; failure position in trajectory; inter-annotator agreement

## Key Results

| Finding | Detail |
|---|---|
| Short/mid-horizon | Strong performance across evaluated models |
| Long-horizon | Sharp performance degradation — "task mirage" |
| Primary failure modes | Context management and planning errors dominate |
| Failure position | Failures cluster at mid-to-late trajectory stages |
| Judge reliability | κ=0.61 IAA, κ=0.84 human alignment |

- **LLM agents perform strongly on short- and mid-horizon tasks but break down sharply on long-horizon tasks** — the "long-horizon task mirage" describes high short-task performance creating false confidence about long-task capability
- Failure attribution reveals that **context management errors and planning failures** are the dominant failure modes at long horizons, not tool availability or grounding accuracy
- Failures **cluster at mid-to-late trajectory stages**: agents typically begin tasks successfully but accumulate errors that compound past a critical horizon length
- The HORIZON Leaderboard enables continuous community benchmarking as models improve, avoiding the contamination risk of static benchmarks

## FoxBrain Relevance
HORIZON directly addresses FoxBrain's most critical deployment challenge: long-horizon enterprise workflows (multi-step procurement processes, complex engineering change orders, multi-document compliance reviews) are precisely the tasks where FoxBrain is most likely to silently fail. The failure attribution framework — categorizing failures by type and trajectory position — should be adopted as FoxBrain's standard agent QA methodology. The finding that failures cluster at mid-to-late stages suggests FoxBrain agents on long workflows need explicit mid-task checkpoints and human review gates, not just end-of-task validation. The κ=0.84 human-aligned LLM-as-judge design provides a scalable template for FoxBrain's internal agent monitoring infrastructure.
